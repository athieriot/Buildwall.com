Help

This buildwall is able to show informations coming from a continuous integration server on a screen. It works by installing a plugin into your integration server, configuring it to send the information you want to the BuildWall. The BuildWall listen to the plugin and display information directly.

Gathering the pieces

You need two pieces to use BuildWall :

  • A continuous integration server, we currently provide integration for TeamCity and Hudson.
  • A computer on which you can dedicate the screen to the tools.

  • Green : You're good to go.
  • Red : Build Failed
  • Yellow : Build currently running.

Build are displayed from right to left, and top to bottom.

The wall can be terminated by issuing a click in the interface.

Interacting Manually with the wall

To check if the buildwall works you can trigger a build result by issuing directly a http command.

Issuing a

curl -X get http://www.buildwall.com/1/abcdefgh/publish?project=foo&status=failed

Optionally you can add a category and/or authors

curl -X get http://www.buildwall.com/1/abcdefgh/publish?project=foo&status=failed&category=trunk&authors=Barnabe

If multiple authors are present, they must be separated with a comma

curl -X get http://www.buildwall.com/1/abcdefgh/publish?project=foo&status=failed&category=trunk&authors=Barnabe,Dorothe
